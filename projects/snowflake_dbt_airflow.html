ELT - Snowflake DBT Airflow

  • Tech Stack: Python, SQL, Snowflake, DBT, Airflow, Docker, Bash
  • Github URL: Project Link
  • DAG: Airflow DAG

This project builds an ELT pipeline using Snowflake, dbt, and Airflow to process and transform Snowflake Sample Data (TPC-H benchmark) into meaningful insights. The pipeline uses batch processing to handle approximately 1 million rows, focusing on orders and lineitem tables. Data is ingested, staged as Views in Snowflake, and transformed through dimension modeling.

The transformation involves creating an order_item table, summary tables with user-defined functions, and a fact table. Validation tests ensure data quality by checking for uniqueness, non-null values, and adherence to business rules (e.g., valid discount and date ranges). The processed data is served to stakeholders like data analysts, machine learning engineers, and data scientists.

Airflow orchestrates the pipeline, running DAGs daily and visualizing the data flow. The setup uses Astro for local Airflow development, with dbt managing models and tests, and Snowflake hosting the data ecosystem. This project emphasizes modular, efficient, and scalable data processing workflows.
